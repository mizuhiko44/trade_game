# UI設計仕様書（Trade Game MVP+）

## 1. 目的
- 画面レイアウト・コンポーネント構成・状態表示ルールを明確化し、
  デザイナー/実装者が同じ指示書で改修できるようにする。

## 2. 共通ルール
- 画面余白: `padding: 20`
- セクション間余白: `gap: 10〜12`
- 見出し: `fontSize: 20〜24`, `fontWeight: 700`
- エラー: 赤 (`#dc2626`)
- ターン状態/通知: 青 (`#1d4ed8`)

## 3. 画面別レイアウト

### 3.1 Home
上から順に配置:
1. タイトル
2. ユーザ情報（名前、アバター、LP）
3. ボーナス受取ボタン
4. CPU戦開始ボタン
5. Debug遷移ボタン

### 3.2 Battle
上から順に配置:
1. タイトル
2. エラー/ターン状態/通知
3. 対戦メタ情報（Match ID、現在価格、ターン、目標価格）
4. 自分/相手アバター
5. チャート種別切替（ドル円/株/債券）
6. 投入金額選択（50/100/200/300）
7. 注文方式（成行/指値）＋指値入力
8. ローソク足チャート
9. 売買・Hold・Item・決済ボタン

### 3.3 Result
上から順に配置:
1. タイトル
2. 勝者サイド / 最終価格
3. ターンサマリ（ターン数、終値レンジ、最終終値）
4. 報酬
5. ホーム戻るボタン

### 3.4 Debug
上から順に配置:
1. タイトル
2. 説明文
3. 送信テキスト入力
4. 送信 / 更新 / 全削除ボタン
5. メッセージ一覧（時刻、source、本文）

## 4. 主要コンポーネント仕様

### CandlestickChart
- 入力: `turns`, `executions`
- 仕様:
  - OHLCからwick/body描画
  - execution marker表示（B/S）
  - markerタップでポジション詳細ポップアップ表示
  - OPEN + MARKET の場合は決済ボタン表示
- 改修ポイント:
  - 色テーマ
  - 横スクロール対応
  - 軸ラベル追加

## 5. ターン制御仕様（Battle）
- デフォルト: 自分ターン（60秒）
- 1秒ごとに残り秒数を更新
- 60秒無操作で自動パス（Hold送信）
- その後、相手ターン表示（5秒）→自分ターンへ戻る

## 6. レイアウト修正指示テンプレート
以下をIssue/PRにそのまま記載して依頼可能:

```md
### UI修正対象
- 画面: Battle
- セクション: 注文方式エリア

### 変更内容
- 指値入力欄をチャートの上に移動
- ボタン間余白を8→12へ
- ターン通知テキストを太字に

### 受け入れ条件
- 既存機能（売買/Item/決済/60秒パス）が壊れない
- iOS/Androidで崩れない
```
