# CPU対戦 面白さ拡充構想（ドラフト）

最終更新: 2026-04-04

## 1. 方針

当面は「他ユーザ対戦（PvP）」より前に、CPU対戦のゲーム体験を強化する。
狙いは以下の3点。

1. **毎試合の展開が変わる**（単調な同型試合を減らす）
2. **判断の納得感がある**（CPUの行動理由が説明可能）
3. **リプレイしたくなる報酬/進行**（短時間でも継続プレイ動機を作る）

## 2. 現状整理

- CPU行動は `decideCpuAction` に集約され、難易度別に売買/ホールド/アイテムを選択している。
- 対戦解決は `resolveTurn` で行い、監査ログに CPU の判断理由も記録できる。
- 勝敗条件は `PNL_BATTLE`（デフォルト）と `TARGET_REACH` の2モードが存在する。

## 3. 面白さ拡充の柱

### 3.1 CPUキャラクター化（人格/戦術タイプ）

難易度（Easy/Normal/Hard）とは別に、CPUの戦術タイプを追加。

- **トレンド追随型**: ブレイク方向に強く追随
- **逆張り型**: 過熱時に反転狙い
- **終盤勝負型**: ラスト2ターンで攻勢
- **防御型**: Shield活用比率が高い

期待効果:
- 同じ難易度でも試合の読み味が変わる
- 「今日はどのCPUが来るか」で再戦性が増える

### 3.2 試合中イベント（ランダム要素）

各試合で 1〜2回、軽い相場イベントを発生。

- 例: `雇用統計ショック`（ボラ上昇）、`要人発言`（上/下バイアス）
- 影響は強すぎない範囲に制限（理不尽回避）

期待効果:
- 中盤の停滞打破
- リプレイ時の展開差分増加

### 3.3 目標達成ミッション（メタ進行）

1試合単位の勝敗以外に、短期ミッションを付与。

- 例: `3ターン連続プラス`, `アイテム未使用で勝利`, `逆張り1回成功`
- ミッション達成で追加コイン/称号ポイント

期待効果:
- 負け試合でも次に繋がる意味を作る
- 上級者向けのやり込み導線を追加

### 3.4 リザルト演出と振り返り

結果画面に「なぜ勝った/負けたか」を短文で表示。

- CPUの決め手行動（例: 終盤Price Spike）
- 勝敗を分けたターン（損益寄与が最大のターン）

期待効果:
- 納得感と学習体験
- 次試合の改善行動に繋がる

## 4. 実装ロードマップ（CPU対戦優先）

### Phase 1（1〜2週間）: 低コストで体験差分を作る

- CPU戦術タイプの追加（2種類から開始）
- 現行ログに「戦術タイプ」タグを追加
- リザルト画面へ簡易サマリ文言追加

**受け入れ基準**
- 同一難易度でも戦術違いで行動分布に差が出る
- 対戦ログからCPUタイプ別の勝率が可視化できる

### Phase 2（2〜3週間）: プレイ継続性の強化

- 試合内ランダムイベント導入
- デイリーミッション3種導入
- ミッション報酬の最小実装

**受け入れ基準**
- 連続5試合で展開パターンの重複率が目に見えて低下
- ミッション達成率の計測が可能

### Phase 3（3週間〜）: 調整・運用基盤

- バランス係数の設定化（DB or 管理ファイル）
- A/Bパラメータテスト（例: イベント発生率）
- KPIダッシュボード整備

**受け入れ基準**
- ノーコードで係数調整できる
- 勝率/継続率/KPIの週次比較が可能

## 5. KPI（面白さ判定）

- **再戦率**: 1日内の2試合目移行率
- **平均試合数**: 1セッション当たりCPU戦回数
- **試合多様性指数**: 行動シーケンス重複率の逆数
- **納得感指標（簡易）**: リザルト後の即離脱率

## 6. 直近タスク（次スプリント）

1. CPU戦術タイプ `trend_follower` / `contrarian` を追加
2. 監査ログへ `cpuStyle` フィールド追加
3. 結果画面に「勝敗を分けたターン」1行サマリを追加
4. 難易度×戦術タイプの簡易シミュレーションを50試合実施

## 7. リスクと対策

- **リスク:** ランダム要素で理不尽感が増える
  - **対策:** 影響幅に上限を設け、表示テキストで事前に通知
- **リスク:** CPUが強すぎ/弱すぎに偏る
  - **対策:** 難易度ごとの勝率目標帯（例: Easy 35〜45%）を定義
- **リスク:** 施策増加で原因追跡が難しくなる
  - **対策:** ログ項目を固定スキーマ化し、変更時は必ずバージョン付与

## 8. 備考

PvP構想は並行で保持するが、実装優先順位は CPU対戦強化を先行する。
CPU体験の改善は、そのままPvP時代の観戦/解説/イベント機能にも再利用できる。
